## Commands

### Backend (Flask)

```bash
# Quick start (creates venv, installs deps, starts Flask dev server)
cd api && ./dev.sh

# Manual setup
cd api
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-2026.txt

# Run dev server
cd api/src && flask run

# Generate config.json (required before first run)
cd api/src/config && python config.py dev

# Run tests
cd api/test && pytest

# Run a single test file
cd api/test && pytest test_auth.py

# Run a single test
cd api/test && pytest test_auth.py::test_login
```

### Frontend (Next.js)

```bash
cd frontend
npm install
npm run dev          # Dev server on :3000
npm run build
npm run lint
npm run format       # Auto-fix with Prettier
npm run format:check
```

## Architecture

### Overview

Full-stack hackathon registration and management platform. Flask REST API backend, Next.js frontend. `frontend-old/` is the frontend from 2025 -- ONLY reference it if the user needs to migrate changes,
reference design decisions, etc. from last year. `frontend/` is target frontend.

### Backend (`api/`)

**Entry point:** `api/src/app.py` — `create_app()` factory that loads `config/config.json`, initializes extensions (JWT, MongoDB, Mail, Slack, Discord), and registers blueprints.

**Blueprint layout** — each feature is a standalone file in `api/src/`:

| File | Prefix | Purpose |
|------|--------|---------|
| `auth.py` | `/api/auth` | Login, JWT issuance, token refresh, logout |
| `accounts.py` | `/api/accounts` | User profiles, resume uploads to S3 |
| `admin.py` | `/api/admin` | Admin-only dashboard endpoints |
| `resumes.py` | `/api/resumes` | Resume retrieval |
| `assign.py` | `/api/judgetool` | Judging tool |
| `announcements.py` | `/api/announcements` | Event announcements |
| `events.py` | `/api/events` | Event management |
| `teammatching.py` | `/api/teammatch` | Team matching algorithm |
| `slack.py` / `discord.py` | `/api/slack`, `/api/discord` | Webhook integrations |

**Database:** `db.py` wraps pymongo with a `MongoDriver` class; a global `db` object is imported by blueprints to access collections.

**Auth:** JWT (Flask-JWT-Extended 3.x). Access token in response body, refresh token in HTTP-only cookie. Protect routes with `@jwt_required` / `@fresh_jwt_required`.

**Config:** `api/src/config/config.json` (gitignored). Generated by `config/config.py dev`. Contains MongoDB URI, JWT secret, mail credentials, AWS keys, Slack/Discord webhook URLs.

**Python version:** 3.9.10 (managed via pyenv — see `api/README.md`).

### Frontend (`frontend/`)

Next.js 16 with App Router (`/app` directory), React 19, TypeScript, Tailwind CSS 4.

**Auth context:** `app/util/auth.tsx` — `AuthProvider` wraps the app and exposes user state. `RequireAuth.tsx` guards protected pages.

**API calls:** Axios to `http://localhost:8000/api/*` in development. JWT access token sent in `Authorization: Bearer` header.

**Key pages:** `/register/login`, `/register/signup`, `/reset_password/[token]`, `/confirm_email/[token]`, and analytics route group `/(analytics)/` (accounts, overview, stats).

### Tests (`api/test/`)

pytest. `conftest.py` provides `client` (Flask test client with per-test DB teardown), `test_db`, and `test_mail` fixtures. Tests connect to a separate `hophacks-test` MongoDB database configured in `test/test.json`.

### CI/CD (`.github/workflows/`)

- **deploy.yml** — triggers on push to `master`; rsyncs to AWS EC2, builds frontend, sets up pyenv/venv, starts backend with gunicorn and frontend with `serve` via tmux.
- **eslint.yml** / **prettier.yml** — run on all branches/PRs; ESLint is zero-warnings enforced.

### External Services

Disable before running locally (see `api/README.md`): AWS S3 (resume uploads), Slack webhook, Discord webhook.
